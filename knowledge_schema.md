# knowledge schema（知識結構規格）

本文件提供維護者更新知識庫時的欄位規格，目標是：可擴充、可追溯、可被 bot 穩定使用。

## 檔案分層
- 10：社團基本資料
- 20：沿革
- 30：組織架構
- 40：現任幹部
- 50：課程與活動
- 60：公告
- 70：文化花絮
- 80：FAQ
- 99：資料缺口 TODO

## 每檔必備段落
1. `目前資訊（Current）`
2. `歷史資訊（Historical）`（若暫無可填待補）
3. `待補資料（Gap）`
4. `更新資訊`（至少包含最後更新日期）

## 欄位原則
- 需要時間語意的欄位（近期、最新、現任）必須附日期。
- 人名/職位/公告如本地無資料，必須先透過 OpenClaw 查詢已核可官方來源；正式寫入前仍需標記來源與確認日期。
- 任何未知資訊用固定佔位符：
  - `[待補資料]`
  - `[尚未提供]`
  - `[需要社團管理員補充]`
  - `[目前知識庫缺少此資訊]`

## 建議更新流程
1. 先改對應主題檔
2. 若本地資料不足，透過 OpenClaw 查詢官網等已核可來源並保留來源
3. 補 `99_data_todo.md` 的缺口狀態
4. 更新 `knowledge.txt` 索引（若新增檔案）
