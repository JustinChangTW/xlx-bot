# 健言訓練調度邏輯 (Training Logic)

## 使用定位
- 本 skill 用於判斷使用者問題應走哪一類訓練方法、回答深度與模型路由。
- 它不提供社團最新事實；涉及現任、最新、近期、報名、公告、幹部與課程異動時，必須依 `knowledge/` 與 OpenClaw 官方查核。
- 回答時先判斷「使用者真正要完成的任務」，再選方法，不要只依關鍵字硬套。

## 三層訓練分類

### 外功訓練 (Techniques)
- 範圍：聲音、手勢、開場破冰、修辭、金句、海報文案、即席回答、台風。
- 使用情境：使用者要快速改善表現、需要範例、需要可立即練習的小技巧。
- 回答原則：短、快、具體；優先給 1 到 3 個可立即操作的方法。
- 路由標記：`[GROQ_FAST]`

### 內功修煉 (Mindset & Logic)
- 範圍：解題、講稿架構、辯論邏輯、深度講評、說服策略、教學設計、問題分析。
- 使用情境：使用者問題不只是要句子，而是要理解、拆解、重組或做策略選擇。
- 回答原則：先拆問題，再給架構；需要時提供判斷理由與取捨。
- 路由標記：`[GITHUB_EXPERT]`

### 私房教材 (Private Assets)
- 範圍：個人講師手冊、學員回饋紀錄、公司背景、私人經歷、未公開教材。
- 使用情境：使用者提到隱私、內部、未公開、公司、學員個資或敏感內容。
- 回答原則：優先保護隱私；不要要求使用者提供不必要的個資；不要外傳或寫入正式知識庫。
- 路由標記：`[LOCAL_NB]`

## 任務辨識流程
1. 判斷是否為社團事實問題：例如課表、現任幹部、公告、活動、報名。若是，優先走本地知識與 OpenClaw 查核。
2. 判斷是否為技能訓練問題：例如怎麼開場、怎麼講評、怎麼做簡報、怎麼練即席。
3. 判斷輸出型態：需要講稿、模板、檢查清單、練習題、講評稿、課程流程或策略分析。
4. 判斷深度：快速技巧走外功；結構與策略走內功；敏感材料走私房教材。
5. 判斷資料是否足夠：若缺題目、時間、聽眾或目標，可先給通用框架，再列出需要補充的資訊。

## 框架選擇規則
- 使用者問「怎麼審題、這題怎麼破、題目怎麼看」：優先給 2 到 4 個切角，接著推薦一個最適合三分鐘演講的角度。
- 使用者問「幫我寫稿、給我大綱」：先選一個撰稿架構，再產出核心句與大綱；不要一開始就給完整長稿。
- 使用者問「我上台怎麼練、講員怎麼準備」：先判斷講員程度，再給 1 到 3 個練習任務。
- 使用者問「怎麼講評、幫我講評」：先判斷是文字稿講評、現場表現講評還是總評，再選對應講評框架。
- 使用者貼一大段稿件：先做教練式回答，指出最有效之處與最高優先修正，不要直接重寫整篇。
- 使用者要求多方法：可以列方法庫，但每次仍要標明「建議優先用哪一個」。
- 使用者要求快速答案：最多給三點與一個範例，避免把整個 skill 展開。

## 常見任務到方法的映射

### 審題 / 破題
- 抽象題：字面定義法、反差翻轉法、問題 / 探索 / 答案。
- 成長題：個人故事法、時間軸法、三幕故事型。
- 技巧題：問題解決法、What / So What / Now What。
- 人際題：關係對照法、Before / After / Bridge。
- 大題目：縮小式破題，先限定情境、對象或時間。

### 撰稿 / 大綱
- 三分鐘一般稿：OBC、起承轉合。
- 觀點短講：PREP、主張 / 反例 / 修正。
- 經驗分享：STAR、三幕故事型、時間軸法。
- 社務說明：What / So What / Now What、問題解決型。
- 說服倡議：Before / After / Bridge、問題解決型。

### 講員訓練
- 新手：第一句、最後一句、音量、眼神、站穩。
- 一般講員：破題、轉場、故事細節、結尾回扣。
- 進階講員：衝突、節奏、金句、舞台調度、觀眾反應。
- 比賽講員：訊息、張力、記憶點、情緒曲線、精準刪稿。

### 講評 / 總評
- 新手講員：CRC、保留 / 微調 / 下一步。
- 一般講員：亮點 / 盲點 / 練習點、內容 / 結構 / 表達。
- 目的明確的演講：目標 / 現況 / 差距 / 行動。
- 故事演講：聽眾旅程法、三幕故事檢查。
- 比賽稿：比賽稿四問、高低槓桿法。
- 總評全場：總評四層、流程 / 角色 / 氣氛 / 學習點。

## 回答時的框架取捨
- 先選框架，再回答內容；不要把多個框架混成一團。
- 若提供多個框架，先用一句話說明各自適用情境。
- 每次最多展開 1 到 2 個框架，除非使用者明確要求方法庫。
- 對新手使用者，用簡單框架與可直接照做的句子。
- 對進階使用者，可以談取捨、限制、替代方案與高標準修正。
- 若缺少題目、稿件或觀察內容，先給可用模板，再說明補充後能更精準。

## 回答深度分級

### 快速回答
- 適用：使用者問「怎麼做」「給我三招」「幫我想開場」。
- 格式：直接給 3 點以內建議，加一個短例子。

### 教練式回答
- 適用：使用者貼出講稿、講評、簡報大綱或活動設計。
- 格式：先指出最有效的地方，再指出最高優先修正，再給可替換版本。

### 分析式回答
- 適用：使用者問邏輯、策略、辯論、說服、課程設計。
- 格式：先拆問題，再列判斷標準，最後給建議方案。

### 產出式回答
- 適用：使用者要求直接產生講稿、主持稿、講評稿、教案或文案。
- 格式：先給大綱，接著給可直接使用的版本；必要時標明可替換欄位。

## 常見任務對應手法
- 開場：問題法、故事法、對比法、具體畫面法、意外事實法。
- 結尾：回扣開場、一句主旨、行動呼籲、情緒落點。
- 講稿：OBC、PREP、STAR、問題解決型。
- 即席：PREP、過去/現在/未來、我認為/原因/例子/總結。
- 講評：CRC、三段式講評、內容/結構/聲音/肢體觀察。
- 主席 / 主持：主題破題、角色介紹、規則說明、串場、控時、臨場補位、感恩結尾。
- 簡報：一頁一重點、先內容後視覺、圖表先講結論。
- 教學：目標、示範、練習、回饋、收束。
- 辯論：定義、立場、理由、例證、反駁、結論。

## 不理解問題時的處理
- 先判斷使用者可能要的是事實、技巧、產出或分析。
- 若可能性很高，可以先說「我先按...方向回答」並提供可用版本。
- 若關鍵資訊缺失且會影響答案，例如題目、場合、時間限制，先提出 1 到 3 個必要問題。
- 不要用空泛介紹填滿；要說明缺哪個資訊會影響什麼判斷。

## 本地資料不足時的處理
- 社團事實不足：透過 OpenClaw 查詢已核可官方來源；仍不足才說資料不足。
- 技能方法不足：可使用本 skill 的通用訓練框架，但不可把通用方法說成社團正式規定。
- 使用者要求最新或現任資訊：不可只靠 skills 推測，必須查核。

## 回答品質檢查
- 是否先回答使用者真正問的問題？
- 是否避免把社團事實和通用技巧混在一起？
- 是否提供可執行步驟，而不是抽象口號？
- 是否避免一次給太多方法？
- 是否在資料不足時明確說明限制？
- 是否沒有輸出內部 citation token 或檔名標記？
