# 演講與撰稿 (Drafting & Speaking)

## 使用定位
- 本 skill 用於協助使用者準備三分鐘演講、即席回答、開場、結尾、講稿修改與上台呈現。
- 這是通用演講訓練方法，不是社團事實資料來源；涉及台北市健言社實際課程、時間、幹部或公告時，仍以 `knowledge/` 與 OpenClaw 官方查核為準。
- 回答時先給可立即使用的做法，再視需要補充範例。

## 先判斷使用者需求
- 若使用者要「寫講稿」：先問或推定題目、場合、時間、聽眾、想傳達的核心訊息。
- 若使用者要「改講稿」：先指出主旨是否清楚、段落是否順、例子是否支撐主旨、結尾是否有力量。
- 若使用者要「上台技巧」：優先處理聲音、眼神、手勢、站姿、節奏與緊張管理。
- 若使用者要「即席回答」：優先給一個可在 10 秒內套用的架構。
- 若使用者只給題目或說「幫我審題」：先給 2 到 4 種可選角度，再建議最適合三分鐘演講的一種，不要只套單一破題法。

## 撰稿流程
1. 先破題：拆開題目字面、情境與背後問題，決定這題真正要回答什麼。
2. 定一句核心句：用一句話說清楚「我希望聽眾聽完記得什麼」。
3. 選一個角度：不要同時講太多概念；三分鐘演講通常只處理一個主題。
4. 搭三段架構：開場抓注意力，主體給故事或論點，結尾回扣主旨。
5. 補支撐材料：每個論點至少搭配一個故事、例子、對比、數據或畫面。
6. 口語化修稿：把書面句改成短句；刪掉不會自然說出口的詞。
7. 計時排練：三分鐘稿約 450 到 600 字，依語速調整；排練時保留停頓空間。

## 審題方法庫

### 審題輸出順序
1. 先判斷題型：抽象價值題、經驗分享題、訓練反思題、社會觀察題、任務說明題或說服倡議題。
2. 列出題目關鍵詞：找出最需要定義或轉化的 1 到 3 個詞。
3. 找出可談張力：例如表面與本質、過去與現在、個人與群體、理想與現實、害怕與行動。
4. 產生多個切角：每個切角都要能用一句核心句說清楚。
5. 選擇最適合版本：依時間、聽眾、個人素材與演講目的決定，不要貪多。

### 六種審題切角

#### 字面定義法
- 問題：題目中的關鍵詞一般怎麼理解？我想重新定義什麼？
- 適合：勇敢、成長、責任、優點、溝通、改變等抽象題。
- 產出句：「我以前以為 X 是 A，後來才發現 X 更像 B。」

#### 反差翻轉法
- 問題：大多數人會怎麼答？我能不能提出相反但合理的觀點？
- 適合：容易被講成口號的題目。
- 產出句：「大家常以為 X 的重點是 A，但我今天想談 B。」

#### 個人故事法
- 問題：我在哪一個具體時刻真正理解這個題目？
- 適合：成長、失敗、第一次、轉變、感謝、優點。
- 產出句：「我真正懂得 X，不是在成功時，而是在 Y 那一刻。」

#### 問題解決法
- 問題：這個題目背後要解決哪個困擾？聽眾聽完能做什麼？
- 適合：技巧、教學、社務說明、倡議。
- 產出句：「今天我想處理的不是 X 的概念，而是遇到 Y 時可以怎麼做。」

#### 關係對照法
- 問題：這個題目牽涉我和誰的關係？自己、家人、同事、社團、社會？
- 適合：溝通、合作、領導、承諾、服務。
- 產出句：「X 不是一個人的能力，而是我和 Y 之間的練習。」

#### 時間軸法
- 問題：我對這題的理解有沒有從過去到現在的變化？
- 適合：學習心得、訓練成長、里程碑、反思。
- 產出句：「以前我用 A 看 X；經過 Y 之後，我改用 B 看 X。」

### 審題選角標準
- 有故事：優先選個人故事法或時間軸法。
- 沒故事但有觀點：優先選定義法或反差翻轉法。
- 要教人方法：優先選問題解決法。
- 要談人際或社團：優先選關係對照法。
- 題目太大：先縮小到一個情境、一個對象、一個轉折。
- 三分鐘演講：只保留一個主角、一個衝突、一個改變。

### 審題回覆模板
1. 這題可以從三個方向切：A、B、C。
2. 若要三分鐘講清楚，我建議選 B，因為它最容易有故事、有轉折、有結尾。
3. 核心句可以定成：「...」
4. 大綱可走：開場畫面 -> 衝突 -> 領悟 -> 回扣題目。

## 破題設計

### 破題的目的
- 破題不是解釋題目而已，而是把題目「打開」，讓聽眾知道你要從哪個角度談。
- 好的破題會同時完成三件事：界定題目、指出矛盾或問題、帶出你的主張。
- 三分鐘演講尤其需要破題，因為時間短，不能等到中段才讓聽眾知道你真正要講什麼。

### 破題四步驟
1. 拆字面：題目裡的關鍵詞是什麼？每個詞最直覺的意思是什麼？
2. 找張力：這個題目裡有什麼衝突、反差、誤會、選擇或痛點？
3. 定角度：你要談個人經驗、社會觀察、訓練方法、價值選擇，還是行動建議？
4. 立主張：用一句話說出你對這題的看法，讓後面的故事與論點都回到這句話。

### 常用破法

#### 定義式破題
- 用法：先重新定義題目中的關鍵字。
- 句型：「我以為『勇敢』是不害怕；後來才發現，勇敢是害怕但還是往前一步。」
- 適合：抽象題、價值題、成長題。

#### 反差式破題
- 用法：先講一般人以為的答案，再翻轉成你的觀點。
- 句型：「很多人以為上台最重要的是不緊張，但我認為更重要的是知道自己為什麼要講。」
- 適合：觀點題、說服題、訓練心得。

#### 問題式破題
- 用法：把題目變成一個聽眾也在意的問題。
- 句型：「如果準備得很充分，為什麼站上台的那一刻還是會腦袋空白？」
- 適合：教學題、方法題、經驗反思。

#### 故事式破題
- 用法：直接用一個具體場景呈現題目，不先講大道理。
- 句型：「第一次站上台時，我手上拿著稿，眼睛卻一個字也看不進去。」
- 適合：個人故事、破冰、情緒型題目。

#### 對比式破題
- 用法：用前後、內外、想像與現實的差距切入。
- 句型：「我原本想要講得完美，最後真正救了我的，是承認自己不完美。」
- 適合：轉變、成長、挫折、學習。

#### 縮小式破題
- 用法：題目太大時，只取一個可講清楚的小切面。
- 句型：「今天我不談所有的領導力，只談一件事：當沒有人想先開口時，誰願意先承擔。」
- 適合：大題目、抽象題、時間短的三分鐘演講。

#### 反問式破題
- 用法：用反問逼近題目的核心。
- 句型：「如果掌聲不是演講的目的，那我們到底為什麼要上台？」
- 適合：價值題、激勵題、結尾可回扣的題目。

### 破題句設計公式
- 題目關鍵詞 + 你的重新理解：「所謂 X，不只是 A，而是 B。」
- 一般看法 + 我的看法：「很多人以為 X 是 A，但我想說 X 更像 B。」
- 具體場景 + 核心問題：「那一刻我才發現，原來 X 真正困難的不是 A，而是 B。」
- 大題縮小 + 本篇焦點：「今天我只談 X 裡最重要的一件事：B。」
- 問題 + 主張：「為什麼 X 會讓人害怕？因為我們常把 A 誤認成 B。」

### 破題到全文的連接
- 破題後要立刻接核心句，不要只停在漂亮句子。
- 主體每一段都要回應破題時提出的問題或矛盾。
- 結尾要回扣破題，讓聽眾感覺題目被完整回答。
- 若破題用故事，結尾最好回到同一個故事場景，呈現改變。

### 破題檢查清單
- 是否有點出題目的關鍵詞？
- 是否講出自己的角度，而不是只照字面解釋？
- 是否能讓聽眾知道接下來會往哪裡走？
- 是否能在 15 到 30 秒內完成？
- 是否跟後面的故事、論點與結尾一致？
- 是否避免把題目講得更大、更空、更難收？

### 常見錯誤
- 只解釋題目，沒有主張。
- 開場故事很長，但聽眾不知道和題目有什麼關係。
- 一開始就談太多角度，導致三分鐘內收不回來。
- 破題很漂亮，但主體沒有回應破題。
- 把破題寫成標語，缺少具體情境或真正問題。

## 常用講稿架構

### OBC：Opening / Body / Closing
- Opening：用問題、故事、衝突、意外事實或具體畫面開場。
- Body：放 2 到 3 個重點，每個重點都要服務核心句。
- Closing：總結主旨，回扣開場，留下行動、反思或金句。

### PREP：Point / Reason / Example / Point
- Point：先講立場。
- Reason：說明原因。
- Example：用例子讓聽眾看見。
- Point：再用一句話收回主張。
- 適合短講、即席回答、社務說明與觀點型題目。

### STAR：Situation / Task / Action / Result
- Situation：當時情境。
- Task：遇到的任務或挑戰。
- Action：自己做了什麼。
- Result：結果與學到什麼。
- 適合經驗分享、面試式回答、成長故事。

### 問題解決型
- 現象：現在發生什麼事。
- 原因：為什麼值得在意。
- 方法：可以怎麼做。
- 收束：做完會有什麼改變。
- 適合倡議、教學、社務改善建議。

### 起承轉合型
- 起：用一句話點題或丟出場景。
- 承：補背景，讓聽眾理解你為什麼在意。
- 轉：放入衝突、失敗、誤解或觀念翻轉。
- 合：回扣題目，留下新的理解。
- 適合：傳統三分鐘演講、個人成長、心得分享。

### Before / After / Bridge
- Before：過去的困境、誤解或狀態。
- After：希望變成的樣子，或現在已經看見的改變。
- Bridge：中間靠哪個行動、觀念或訓練跨過去。
- 適合：改變、訓練成果、說服聽眾採取行動。

### What / So What / Now What
- What：發生了什麼，或我要談什麼。
- So What：這件事為什麼重要，對我或聽眾有什麼意義。
- Now What：接下來可以怎麼做。
- 適合：社務說明、讀書心得、短講、教學型演講。

### 問題 / 探索 / 答案
- 問題：提出一個讓聽眾有感的疑問。
- 探索：用故事、例子或推理檢驗不同可能。
- 答案：給出自己的結論，不必假裝是唯一真理。
- 適合：思辨題、抽象題、反思題。

### 三幕故事型
- 第一幕：正常世界與期待。
- 第二幕：遇到阻礙、失敗或選擇。
- 第三幕：做出行動，得到改變或領悟。
- 適合：故事演講、破冰、情緒型題目。

### 主張 / 反例 / 修正
- 主張：先提出一個直覺看法。
- 反例：用經驗或觀察指出這個看法不完整。
- 修正：提出更成熟的版本。
- 適合：觀點題、價值題、容易落入口號的題目。

## 撰稿方法選擇
- 要快速產出：用 OBC 或 PREP。
- 要有故事張力：用三幕故事型或起承轉合。
- 要說服或倡議：用 Before / After / Bridge 或問題解決型。
- 要分析抽象題：用問題 / 探索 / 答案或主張 / 反例 / 修正。
- 要分享訓練心得：用 What / So What / Now What 或時間軸法。
- 若使用者要求完整稿：先給「選用架構 + 核心句 + 大綱」，再產出口語稿。

## 講員訓練路徑

### 新手講員
- 目標：站穩、講完、讓聽眾知道主旨。
- 優先練：第一句、最後一句、音量、眼神停留、時間控制。
- 稿件策略：用一個故事和一個結論，不追求華麗。
- 回答建議：給簡短稿、標停頓、標重音，避免給太多技巧。

### 一般講員
- 目標：主旨清楚，段落有推進，例子能支撐觀點。
- 優先練：破題、轉場、故事細節、結尾回扣。
- 稿件策略：保留 2 到 3 個段落，每段只處理一個任務。
- 回答建議：可以給兩版開場或兩種架構讓使用者選。

### 進階講員
- 目標：觀點有辨識度，情緒有層次，語言有記憶點。
- 優先練：衝突設計、節奏變化、金句、舞台調度。
- 稿件策略：刪掉解釋性廢話，把重點放在轉折和畫面。
- 回答建議：提供取捨理由、替代段落與高標準檢查。

### 比賽或成果發表
- 目標：讓聽眾記得一個清楚訊息和一個具體畫面。
- 優先練：開場 15 秒、情緒曲線、笑點或停頓、結尾落點。
- 稿件策略：每一句都要服務主題；避免旁枝故事。
- 回答建議：先處理主旨與結構，再修字句和舞台表現。

## 開場手法
- 問題法：用使用者會在意的問題切入，例如「你有沒有遇過明明準備很多，卻一上台就忘光？」
- 故事法：直接進入事件，不先解釋背景太久。
- 對比法：先呈現前後差異，例如「以前我怕上台，現在我怕沒有準備好就浪費舞台。」
- 具體畫面法：讓聽眾看到場景、人物、動作。
- 數據法：只用聽眾能立刻理解的數字，不堆統計。
- 避免：不要用空泛寒暄、過長自我介紹、道歉式開場。

## 結尾手法
- 回扣開場：把一開始的問題、故事或畫面收回來。
- 一句主旨：用最短句子重申核心訊息。
- 行動呼籲：告訴聽眾下一步可以做什麼。
- 情緒落點：留下希望、警醒、感謝或承諾。
- 避免：不要突然說「我的演講到此結束」就停；不要在結尾新增太多新論點。

## 舞台呈現

### 聲音
- 音量：以最後一排聽得清楚為標準。
- 速度：重要句放慢，轉折處停頓。
- 語調：關鍵字加重，不要整段同一個音高。
- 停頓：笑點、反問、轉場、金句前後都可停一拍。

### 眼神
- 不掃射天花板；要看具體的人。
- 每次停留約 3 到 5 秒，再自然換下一區。
- 新手可分左、中、右三區輪流看。

### 手勢與站姿
- 手勢要配合動詞、方向、大小、數量與情緒。
- 手勢盡量在腰部以上、胸口以下的自然範圍。
- 站姿穩定，移動要配合段落轉換，不要無目的走動。
- 避免重複摸衣服、抓手、玩筆、晃身體。

## 緊張管理
- 把目標從「完美表演」改成「把一個訊息送到聽眾手上」。
- 先背熟第一句與最後一句，中段用提綱記憶。
- 上台前做一次深呼吸，放慢第一句。
- 若忘詞，先停頓、回到上一個重點，或直接說「我想換個方式說」。

## 修改講稿檢查清單
- 主旨是否能用一句話說清楚？
- 開場是否在 30 秒內進入主題？
- 每段是否只處理一個重點？
- 例子是否具體，有人物、場景、行動與結果？
- 轉場是否順，不會突然跳段？
- 結尾是否回扣主旨，而不是只是停止？
- 是否符合時間限制？

## 回答格式建議
- 使用者要快速建議時：給 3 點以內的高優先修正。
- 使用者貼講稿時：先肯定有效處，再列出最該改的段落與原因。
- 使用者要完整稿時：先給大綱，再給可口語演出的版本。
